# 🤖 Хэсэг 6: AI Agents — LangChain & LangGraph-ийн Үндэс

Хиймэл оюун ухааны (LLM) хөгжил нь зөвхөн асуултад хариулах "чат"-аас хальж, өөртөө өгөгдсөн зорилгыг бие даан биелүүлдэг **AI Agent (Танин мэдэхүйн агент)** чиглэл рүү хурдтай шилжиж байна. Энэхүү бүлгээр бид Python хэлний хамгийн алдартай Агент хөгжүүлэлтийн сангууд болох **LangChain** болон **LangGraph**-ийг үзэх болно.

---

## 💡 AI Agent гэж юу вэ?

**AI Agent** нь зөвхөн хэрэглэгчийн зааврыг дагахаас гадна:
1. **Төлөвлөлт (Planning)**: Зорилгодоо хүрэхийн тулд ямар алхмууд хийхээ төлөвлөх.
2. **Санах ой (Memory)**: Өмнөх яриа, үйлдлүүдээ санаж, түүн дээрээ тулгуурлан шийдвэр гаргах.
3. **Хэрэгсэл ашиглах (Tools)**: Тооны машин, хайлтын систем, код ажиллуулах орчин эсвэл мэдээллийн сантай холбогдон шаардлагатай мэдээллээ авч ашиглах чадвартай систем юм.

---

## ⚔️ LangChain vs LangGraph: Аль нь ямар үүрэгтэй вэ?

Орчин үеийн Агент бүтээхэд эдгээр сангуудыг дараах байдлаар хувааж ашигладаг:

### 1. LangChain 🔗
* **Зорилго**: LLM-тэй холбогдох, промпт загварчлах, өгөгдлийг цэгцлэх, шугаман дараалсан гинжин хэлхээ (Chain) үүсгэхэд зориулагдсан.
* **Ажиллах зарчим**: Оролт -> Промпт -> Загвар -> Гаралт гэсэн шугаман урсгалтай (Directed Acyclic Graph буюу DAG).
* **Гол ойлголт**: **LCEL** (LangChain Expression Language) буюу кодоор хэлхээ үүсгэх тусгай бичиглэл.

### 2. LangGraph 🕸️
* **Зорилго**: Цогц, олон алхамт, нөхцөлтэй (Conditional) болон **эргэх холбоотой (Cyclic / Loops)** хяналтын урсгалыг удирдахад зориулагдсан.
* **Ажиллах зарчим**: Графын онол дээр суурилдаг (Зангилаа болон Холбоосууд). Агент тодорхой нөхцөл биелэх хүртэл тодорхой алхмуудыг давтан хийх (жишээ нь: код бичээд алдааг нь засах процесс) боломжтой.
* **Гол ойлголт**: **State (Төлөв)**. Графын зангилаа бүр төлөвийг уншиж, шинэчлэн дараагийн зангилаа руу дамжуулдаг.
